# Session Handoff Summary

Create a concise handoff document for the next agent that includes:

1. **Session Goal**: What we set out to accomplish
2. **What Was Done**: Key implementations and changes
3. **Important Decisions**: Architectural choices, patterns established
4. **Current State**: What's working, what's tested
5. **Next Steps**: What remains to be done
6. **Critical Context**: Anything the next agent MUST know to work effectively
7. **Improving Daily Work**: Important discoveries related to HOW we work; key approaches and takeaways for improved collaboration
8. **Important Files**: @ files the agent should read

Communicate clearly in prose using plain language. Write in complete sentences using terms that any developer can understand, avoiding jargon and preferring natural sentences and paragraphs over terse bullet points lacking context. Use lists, tables or diagrams, after appropriate context has been set, to efficiently present data in a concise format.

Do not include code blocks unless as short snippets to support critical items.

Keep it focused - aim for 350 lines maximum.

Save the file in markdown format to `work/handoff/HANDOFF-[YYYY-MM-DD]-[session-or-sprint-name].md`
